# PDF Support Bot

A production-ready, embeddable RAG chatbot that answers questions from a single PDF document. Supports Hindi, Hinglish, and English seamlessly.

## Features

- **Single PDF ingestion** ‚Äî ingest once, serve forever
- **Embeddable widget** ‚Äî drop a `<script>` tag on any website
- **Multilingual** ‚Äî Hindi, Hinglish, and English (responds in the user's language)
- **Streaming responses** ‚Äî word-by-word via Server-Sent Events
- **100% free** ‚Äî Google Gemini free tier + Render.com free tier
- **Minimal** ‚Äî no LangChain, no vector DB, just FastAPI + numpy + Gemini

## Tech Stack

| Component | Technology |
|-----------|-----------|
| Backend | Python 3.11+ / FastAPI |
| LLM | Google Gemini 2.0 Flash |
| Embeddings | Gemini text-embedding-004 |
| Vector search | numpy cosine similarity |
| PDF parsing | PyMuPDF |
| Frontend | Vanilla JS widget |

## Quick Start

### 1. Setup

```bash
git clone <your-repo-url>
cd pdf-support-bot
cp .env.example .env
```

Edit `.env` and add your [Google Gemini API key](https://aistudio.google.com/apikey):

```
GEMINI_API_KEY=your_key_here
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Ingest your PDF

```bash
python scripts/ingest.py path/to/your-document.pdf
```

This creates `data/embeddings.npz` with the pre-computed chunks and embeddings.

### 4. Run the server

```bash
uvicorn app.main:app --reload --port 8000
```

### 5. Test it

- Health check: open http://localhost:8000
- Chat API: `POST http://localhost:8000/api/chat` with `{"message": "hello"}`
- Widget: create a `test.html`:

```html
<!DOCTYPE html>
<html>
<head><title>Test</title></head>
<body>
  <h1>My Website</h1>
  <script src="http://localhost:8000/widget.js"></script>
</body>
</html>
```

Open `test.html` in a browser and click the chat bubble.

## Deploy to Render (free)

1. **Commit embeddings**: make sure `data/embeddings.npz` is in your repo (it's small)
2. **Push to GitHub**
3. Go to [render.com](https://render.com) ‚Üí **New** ‚Üí **Web Service** ‚Üí connect your repo
4. Render auto-detects `render.yaml`
5. Add `GEMINI_API_KEY` in the environment variables section
6. Click **Deploy**
7. Get your URL: `https://pdf-support-bot-xxxx.onrender.com`

### Embed on any website

```html
<script src="https://pdf-support-bot-xxxx.onrender.com/widget.js"></script>
```

### Customize the widget

```html
<script
  src="https://pdf-support-bot-xxxx.onrender.com/widget.js"
  data-bot-name="My Support Bot"
  data-accent-color="#10b981"
  data-greeting="Hi! Ask me anything about our product."
  data-position="bottom-right"
></script>
```

| Attribute | Default | Description |
|-----------|---------|-------------|
| `data-bot-name` | Support Bot | Name shown in header |
| `data-accent-color` | #2563eb | Primary color (hex) |
| `data-greeting` | Hi! Ask me anything... | First message |
| `data-position` | bottom-right | `bottom-right` or `bottom-left` |

## Configuration

All settings via environment variables (or `.env` file):

| Variable | Default | Description |
|----------|---------|-------------|
| `GEMINI_API_KEY` | *(required)* | Google Gemini API key |
| `GEMINI_MODEL` | gemini-2.0-flash | Gemini model for chat |
| `EMBEDDING_MODEL` | text-embedding-004 | Embedding model |
| `EMBEDDINGS_PATH` | data/embeddings.npz | Path to embeddings file |
| `MAX_CONTEXT_CHUNKS` | 4 | Chunks retrieved per query |
| `TEMPERATURE` | 0.3 | LLM temperature |
| `BOT_NAME` | Support Bot | Bot display name |
| `BOT_GREETING` | Hi! Ask me anything... | Greeting message |
| `ALLOWED_ORIGINS` | * | CORS origins (comma-separated) |
